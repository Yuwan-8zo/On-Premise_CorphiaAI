# UI 視覺與體驗優化分析報告

根據您提供的截圖，我對目前的 UI 進行了詳細的視覺審查。目前的畫面已經具備了很好的現代化骨架，但在**對比度、層級感（Elevation）與細節質感**上還有很大的優化空間。

以下是我認為必須進行優化的核心痛點，分為五大方向：

## 1. 致命問題：對比度與文字可讀性 (Contrast & Legibility)
> [!WARNING]
> 這是目前最嚴重的問題，直接影響了產品的可用性。

- **AI 回應文字過暗**：主聊天區的 AI 回覆文字（如「這是一個模擬的 AI 回應...」）使用了過深的灰色或藍灰色，在 `#181816` 的深色背景下幾乎無法閱讀。
  - **解決方案**：必須將這些內容文字全面替換為 `--text-primary` (亮白色) 或 `--text-secondary` (淺灰色)，以符合企業級 UI 的無障礙閱讀標準。
- **內嵌標籤與程式碼區塊**：`[注意]` 標籤的底色與字色需要重新調配，目前的顏色搭配顯得有些混濁；行內程式碼（如 `ollama run ...`）的白底黑字雖然清楚，但可以考慮改用帶有一點品牌色調的淺棕色背景，會更融合整體風格。

## 2. 側邊欄 (Sidebar) 的層次與精緻度
- **缺乏物理邊界感**：側邊欄與主畫面之間目前只靠非常些微的背景色差（`#202022` vs `#181816`）來區分，導致畫面看起來有些扁平。
  - **解決方案**：在側邊欄右側加上一條極細的邊框 `border-r border-border-subtle`，或者加入微弱的右側陰影，將側邊欄在 Z 軸上「抬高」，增加 3D 空間感。
- **「一般 / 專案」切換膠囊**：目前的切換開關整體偏暗，未選取狀態的文字對比度稍弱。可以提升選取方塊的亮度或加上非常輕微的 `box-shadow` 讓它看起來像實體的按鈕。
- **清單 Hover 效果**：對話列表（如「新對話」）的 Hover 狀態需要更明顯的回饋，建議使用 `hover:bg-bg-elevated` 配合細微的文字提亮。

## 3. 頂部導覽區塊 (Header / Top Bar)
- **模型選擇器 (Loading Models...) 像純文字**：右上角的模型狀態與下拉選單，目前沒有按鈕的外框或背景，看起來就像漂浮的純文字，使用者很難意識到那是可以點擊切換模型的按鈕。
  - **解決方案**：將其包裝成一個 Pill (膠囊狀) 按鈕，加上 `bg-bg-surface border border-border-subtle`，並加上向下的箭頭圖示 (Chevron Down)。
- **標題列太空曠**：左上角只有單純的 "Corphia" 文字，可以考慮加入專案專屬的 Logo Icon，或者將目前所在的「對話標題」置中顯示。

## 4. 輸入框與操作區 (Chat Input Area)
- **送出按鈕缺乏指引性**：右側的「↑ (送出)」按鈕背景目前是深灰色。當使用者輸入文字後，這個按鈕應該要亮起（變成品牌主色 Bronze `bg-accent`），提供強烈的「可發送」視覺暗示。
- **輸入框質感**：目前的輸入框比較平實。可以加入Focus狀態的發光效果 (`focus:ring-1 focus:ring-accent/50`)，讓打字時的專注感更強。左側的 ✨ (Sparkle) 按鈕可以考慮加上一點玻璃透視效果或漸層，突顯 AI 屬性。

## 5. 訊息排版與氣泡 (Message Bubbles)
- **缺少對話氣泡的包覆感**：目前的訊息（無論是使用者還是 AI）都直接印在背景上，這種設計雖然極簡，但對於長篇 AI 回覆容易顯得散亂。
  - **解決方案**：考慮給使用者的訊息一個淡棕色 (`bg-accent/10`) 的底色氣泡，而 AI 訊息保持透明背景但增加左側的視覺引導線，或者反過來，增強雙方的對話來回感。
- **行距與段落間距**：目前 AI 的多段落文字之間有些擁擠，需要增加 `mb-4` 與 `leading-relaxed` (1.625 行高)，讓長篇文章的閱讀呼吸感更好。
